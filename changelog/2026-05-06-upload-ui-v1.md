# Upload UI v1

## Summary

Added a simple read-only Registry panel upload interface for sending files to `POST /api/artifacts/upload`, allowing files to enter Project Anam as saved, registered, indexed artifact source material.

## Files Changed

- `frontend/src/App.jsx`
- `frontend/src/components/RegistryPanel.jsx`
- `frontend/src/styles.css`
- `changelog/2026-05-06-upload-ui-v1.md`

## Behavior Changed

- The Registry panel now includes an upload form with a file picker, optional title, and optional description.
- Uploads send multipart form data to `/api/artifacts/upload` with `authority=source_material` and the active user id when available.
- Successful uploads show a compact result summary with artifact title, file path, indexing status, event chunks, and content chunks.
- Unsupported files surface metadata-only indexing clearly without treating it as a failed upload.
- Successful uploads refresh the artifact registry list.
- Manual Registry refresh clears the current upload result and error state.

## Known Limitations

- No drag/drop upload.
- No delete, edit, or artifact manager workflows.
- No source conversation/message linking from the UI in v1.
- No PDF, OCR, Office document, image, audio, or video extraction.
- No manual browser smoke test was run in this patch.

## Project Anam Alignment Check

- Does not assign the entity a name.
- Does not call the entity Anam or Tír.
- Does not assign personality.
- Preserves raw experience by letting files enter as traceable artifacts.
- Treats uploaded files as source material, not operational guidance, identity, personality, or core belief.
- Does not add autonomy, image generation, speech, vision, AI write/action tools, self-modification, or backend behavior changes.
